Remarks

Supported features are: Standard features - prefixed with "http://xml.org/sax/features/"
NameProperties
namespaces Access: read/write, default true.
Should be set *before* parsing is started. If set later, it will only affect the reporting of QNames, that is, if namespaces was true at the beginning, then QName/Prefix reporting can be turned off during parsing, but namespace processing will stay active.
namespace-prefixes Access: read-only, value depends on namespaces feature.
The underlying Expat parser never reports xmlns attributes when namespace processing is turned on, so this feature is set to false in this case even though prefixes (in QNames) would still be reported. Conversely, when namespace processing is turned off then this feature is automatically set to true.
validation Access: read/write, fixed false.
external-general-entities Access: read/write, default false.
external-parameter-entities Access: read/write, default false.
resolve-dtd-uris Access: read/write, default false.
lexical-handler/parameter-entities Access: read/write, default true.
With limitations - See remarks regarding internal entities and ILexicalHandler on ExpatReader. It is not certain that the way this feature is implemented conforms to SAX2, but it works the same way as in MSXML4. It seems that if it is turned off then parameter entities should still be reported if skipped, but this is not done here since internally this feature is equivalent to the parameter-entities extension feature which turns parameter entity parsing on or off.
xmlns-uris Access: read/write, fixed false.
xml-1.1 Access: read-only, fixed false.
unicode-normalization-checking Access: read/write, fixed false.
xml-declaration Access: read-only, fixed true.
use-external-subset Access: read-only, fixed true.
reader-control Access: read-only, default depends on compile options.
Returns if the Suspend()/Resume() functionality is implemented. This is true when compiled with EXPAT_1_95_8_UP defined and version 1.95.8 (or later) of the Expat library is used.
Extension features - prefixed with "http://kd-soft.net/sax/features/"
NameProperties
parameter-entities Access: read/write, default true.
Turns parameter entity parsing on or off. This affects all of the external subset, as well as parameter entities in the internal subset. The rest of the internal subset is still being processed.
skip-internal-entities Access: read/write, default false.
Skip or (silently) expand internal entities (parameter and general).
parse-unless-standalone Access: read/write, default false.
When turned on external parameter entities will not be parsed if the XML declaration specifies standalone = "yes".
standalone-error Access: read/write, default false.
Determines if the parser returns an error, when it encounters an external subset or a reference to a parameter entity, but the document has standalone="yes" in the XML declaration.

Note: "read/write, fixed false" means that the feature can be set, but only to the value false, otherwise an exception will be thrown.
