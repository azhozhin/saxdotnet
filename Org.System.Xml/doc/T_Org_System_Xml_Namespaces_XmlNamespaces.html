Remarks

  • A new namespace scope is activated by calling PushScope. When parsing, this should be done when a start element tag is encountered, but after the namespace declarations for this start tag have been processed. When writing, this should be done just before the start element tag is written out.
  • All prefix mappings must have been added (using AddMapping) to the new namespace scope before it is activated, or in other words, AddMapping calls apply to the next scope to be activated. When parsing, these calls should be made when namespace declarations attributes are processed. When writing, these calls should be made before PushScope is called for the next element's start tag.
  • To leave (deactivate) a namespace scope one must call PopScope, which releases all namespace prefix mappings established in that scope. When parsing, this should be done right after an end element tag was encountered. When writing, this should be done right after the end element tag was written out.
  • This class is designed to use PushScope and PopScope at every level of element nesting - even when no new prefix mappings are declared. This way it can keep track of empty namespace scopes and does not require the programmer to match PopScope calls to PushScope calls at the right level.
  • To declare prefix mappings for the default namespace just use a null prefix, for example: AddMapping(null, "http://myuri").
  • To "undeclare" a prefix mapping just add a mapping with a null URI: AddMapping("myprefix", null). This works for default and regular namespaces, although the latter is *only* legal in the Namespaces for XML 1.1 specification.
  • To re-use an instance for another XML document one must call Reset before calling any other methods.
